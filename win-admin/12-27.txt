powershell for browser automation:
* launch a specific browser with arguments (Chrome, Edge, Firefox)
* open URLs, profiles, custom flags, etc.
* use with frameworks such as Selenium Webdriver "for full control"
* Use CDP/BiDi protocol wrappers (like the GitHub project in the search results) to script browser actions

powershell CANNOT:
* directly inject javascript into a browser tab
* directly control browser extensions
* directly manipulate tampermonkey (however, it can trigger the conditions under which Tampermonkey runs)

Tampermonkey runs userscripts that can:
* modify the page
* automate clicks
*fill forms
*observe DOM changes
*run logic on page load

Automation chain with Powershell and Tampermonkey used together:
Powershell opens browser and loads the url; Tampermonkey script executes after URL loads

Tampermonkey runs INSIDE the browser
Powershell runs OUTSIDE the browser
They interact through:
* URLs
* Query parameters
* Local files
* Browser files
*Custom set triggers

Powershell CANNOT directly interact with a browser for these reasons:
1.) It runs at the OS level, meaning that it simply interacts with Windows commands and processes that tell it to do something.

2.) It can modify webpage files, such as HTML, CSS, and JS, because they are only local files.

3.) Browsers block things like malware, scripts from reading tabs, external code-injecting programs and OS-level DOM manipulation tools for security purposes.

How to merge repos using Git commands-sourced from https://stackoverflow.com/questions/1425892/how-do-you-merge-two-git-repositories:

```
cd path/to/project-a
git checkout some-branch

cd path/to/project-b
git remote add project-a /path/to/project-a
git fetch project-a --tags
git merge --allow-unrelated-histories project-a/some-branch
git remote remove project-a
```
